GestureDetector

Android sdk给我们提供了GestureDetector类，用于辅助检测用户单击、滑动、长按、双击等行为,本文主要讲述GestureDetector的基本使用跟源码解析。

基本使用

GestureDetector这个类对外提供了三个接口和一个外部类：

  • 接口：OnGestureListener、OnDoubleTapListener、OnContextClickListener
  • 内部类:SimpleOnGestureListener

OnContextClickListener：外接键盘鼠标右键(具体没了解)。

OnGestureListener

onGestureListener必须重写以下方法:

  • onDown(MotionEvent e):当手指按下的时候
  • onShowPress(MotionEvent e):当用户手指在屏幕上按下,而且还未移动和松开的时候
  • onSingleTapUp(MotionEvent e):当手指在屏幕上轻轻点击的时候
  • onScroll(MotionEvent e1, MotionEvent e2, float distanceX, float distanceY):当手指在屏幕上滚动的时候
  • onLongPress(MotionEvent e):当手指在屏幕上长按的时候
  • onFling(MotionEvent e1, MotionEvent e2, float velocityX, float velocityY):当手指触摸屏快速滑动后松开的时候,e1代表第一个ACTION_DOWN MotionEvent,最后一个ACTION_MOVE MotionEvent,velocityX代表X轴上的速度,velocityY代表Y轴上的速度

基本使用：
1、实例化OnGestureListener对象，实现其方法
2、实例化GestureDetector对象
3、onTouchEvent(MotionEvent event)中拦截或View的onTouch(View v, MotionEvent event)

OnDoubleTapListener

OnDoubleTapListener必须重写以下方法:

  • onSingleTapConfirmed(MotionEvent e)：当屏幕接收到单击事件，也就是当手指在点击了屏幕之后在一定时间内没有接收到第二次的点击事件
  • onDoubleTap(MotionEvent e)：双击事件
  • onDoubleTapEvent(MotionEvent e)：触发onDoubleTap以后，发生的其它动作

GestureDetector的构造：

创建了一个GestureHandler对象：

可以看出GestureDetector类继承雨Handler，重写handleMessage方法处理消息，消息类型有3种：SHOW_PRESS、LONG_PRESS、TAP：

  • TAP：如果当前手指不是保持down状态，回调onSingleTapConfirmed()方法
  • SHOW_PRESS：回调onShowPress()方法
  • LONG_PRESS：回调onLongPress()方法

之后的init()方法就是一些参数的初始化：

onTouchEvent
变量

范围变量：

  • mTouchSlopSquare; 用来判断是否是单击，移动距离小于mTouchSlopSquare，则是单击，大于，则是onScroll
  • mDoubleTapTouchSlopSquare: 单击移动距离所允许的范围，超过则不构成双击
  • mDoubleTapSlopSquare:第二次按下的位置和第一次按下的位置的距离允许的范围，超过则不构成双击

状态变量：

  • mIsDoubleTapping: 当前事件是否是双击事件
  • mStillDown: 是否按下
  • mAlwaysInTapRegion: 单击是否在在小范围内移动，true:不需要onScrool false:onScroll
  • mAlwaysInBiggerTapRegion: 双击是否在在小范围内移动，true:双击成立 false:双击不成立
  • mDeferConfirmSingleTap：是否延迟确定点击。在handleMessage中消息类型为TAP时，如果手指为非按下状态，则回调onSingleTapConfirmed接口，否则，mDeferConfirmSingleTap置为true，其他情况全部置为false
  • mInLongPress: 是否正在处理长按
  • mIsLongpressEnabled: 是否支持长按
    输入处理
    对输入事件做出统一处理，提取一些共有的信息，如多个点同时触摸时候的中心焦点和滑动速度等，然后根据事件的分类做出相应的处理:

InputEventConsistencyVerifier对输入事件进行的一致性检测的结果并不影响GestureDetector的运行，如果检测到一致性不符合的事件（只有UP事件而前面没有DOWN事件），就只会输出log告诉开发者

ACTION_DOWN
  1. mDoubleTapListener不为null的情况下，如果当前handler中有TAP消息，移除TAP消息，判断是否是第二次点击，如果是第二次点击回调onDoubleTap和onDoubleTapEvent，否则是第一次点击没发送一个延时消息TAP，延时时间DOUBLE_TAP_TIMEOUT；
  2. 重置CurrentDownEvent，标记各种状态
  3. 如果支持长按事件，发送延时消息LONG_PRESS，延时时间mCurrentDownEvent.getDownTime() + LONGPRESS_TIMEOUT
  4. 发送延时消息SHOW_PRESS，延时时间mCurrentDownEvent.getDownTime() + TAP_TIMEOUT
  5. 回调onDown方法
ACTION_MOVE
  1. 正在长按或者点击了鼠标右键，返回
  2. 是双击后的事件，回调onDoubleTapEvent
  3. 之前未移动则判断当前是否已超过点击范围，超过范围回调onScroll、移除相关事件，并标记为已移动
  4. 之前已移动，则继续回调onScroll
