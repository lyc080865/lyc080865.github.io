Androdi build.gradle文件

Android Studio是采用Gradle来构建项目的。Gradle是一个非常先进的项目构建工具，它试用了一种基于Groovy的领域特定语言（DSL）来声明项目设置。官网地址https://developer.android.google.cn/studio/build/

插件版本与Gradle版本对应关系：

插件版本 Gradle版本
1.0.0 - 1.1.3 2.2.1 - 2.3
1.2.0 - 1.3.1 2.2.1 - 2.9
1.5.0 2.2.1 - 2.13
2.0.0 - 2.1.2 2.10 - 2.13
2.1.3 - 2.2.3 2.14.1+
2.3.0+ 3.3+
3.0.0+ 4.1+

当我们创建一个应用时，AndroidStudio默认会有三个gradle文件，分别为：

  • 主目录下build.gradle
  • 主目录下settings.gradle
  • app下build.gradle

接下里我们分别介绍其的作用,目标版本为2.3.3(3.0后有很大改动)。

主目录build.gradle

这个文件是整个项目的gradle基础配置文件

  • buildscript:编译脚本
  • repositories://声明仓库的源，默认是jcenter，之前版本则是mavenCentral，也可以两个center共存。jcenter可以理解成是一个新的中央远程仓库，兼容maven中心仓库，而且性能更优
  • dependencies : //声明一些依赖库，声明gradle版本
  • allprojects:声明了所有project默认的仓库源
  • task clean:task clean声明了一个任务，任务名叫clean（也可以改为其它），任务类型是Delete（也可以是Copy），就是每当修改settings.gradle文件后点击同步，就会删除rootProject.buildDir下的文件(实际上我看到的效果是清除了External Libraries里的包，然后又添加了一次)。

buildscript中的声明是gradle脚本自身需要使用的资源。可以声明的资源包括依赖项、第三方插件、maven仓库地址等。而在build.gradle文件中直接声明的依赖项、仓库地址等信息是项目自身需要的资源。

gradle3.0版本的改动

依赖问题

新建一个Android工程的时候，gradle3.0+中build.gradle文件中的依赖默认为implementation，而不是之前的compile。另外还有依赖指令api。implementation可以提升编译速度.

api与implementation的区别：

  • api完全等同于compile
  • implementation对于使用了该命令编译的依赖，对该项目有依赖的项目将无法访问到该项目中使用该命令编译的其他依赖包的引用，也就是说该依赖包隐藏在该项目内部，不对外部公开，依赖关系不会传递。

例如：当有一个modelu为test，我们引用了gosn依赖包，然后我们在app项目中添加对test的依赖。当使用的是implementation编译时，test可以引用gson，而app不能；当使用的事api编译时，test和app都可以引用。

Google建议依赖首先应该设置为implementation,如果有问题再使用api。
