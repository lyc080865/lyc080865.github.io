Markdown 基本语法

简述

MarkDown 是一种可以使用普通文本编辑器编写的语言标记，通过简单的表示语法，它可以使普通文本内容具有一定的格式。

Markdown官方文档

Markdown语法说明（简体中文版）

常用的语法规则

建议每个符号与文字之间留有一个空格

标题

Markdown支持两种标题的语法，类Setext和atx形式.
类Setext形式是用底线的形式，利用 = （最高阶标题）和 - （第二阶标题），例如：

需要至少两个 = 或 - 才有效果。

类Atx形式则是在行首插入一到6个#，对应标题1到6阶，例如：

1阶为最大，6阶为最小。

段落与换行

  一个Markdown段落是由一个或多个连续的文本行组成，它的前后要有一个以上的空行（空行的定义是显示上看起来是空的，便会被视为空行，比如说，若某一行只包含空格和制表符，则该行也会被视为空行）。普通段落不该用空格或制表符来缩进。

分段：段落之间空一行即可。
换行：只需要在行末加两个空格键和一个回车键即可换行（依赖于Markdown插入<br/>标签实现的，也可以自己手动写入）。

文本样式
  • 加粗：**加粗**
  • 斜体：*斜体*
  • 删除线：~~删除线~~
引用

只要在文本内容之前加大于号（>）即可以将文本变成引用文本,还可以使用多个嵌套引用。

链接

Markdown支持两种形式的链接语法：行内式 和 参考式 两种形式。不管哪一种，链接文字都是用方括号（[]）来标记。

行内式：方块括号后面紧接着圆括号并插入网址链接，如果还想要加上链接的title文字，只要在网址后面，用双引号把title文字包起来即可，格式：[文本内容](链接的地址 “title”)，例如：

参考式：在链接文字的括号后面再接上另一个方括号，而在第二个方括号里面要填入用以辨别链接的标记，然后在文件的任意处，你可以把这个标记的链接内容定义出来；

链接内容定义的形式为：[id]: xxx.xxx "title" （：后面有空格或制表符，title 非必须）
链接网址可以用方括号包起来，链接辨别标签可以有字母、数字、空白和标点符号，不区分大小写。

隐式链接标记功能让你可以省略指定链接标记，这种情形下，链接标记会视为等同于链接文字，要用隐式链接标记只要在链接文字后面加上一个空的方括号，例如：

然后配置 _config.yml文件：

将_config.yml文件中的配置项post_asset_folder设为true后，执行命令$ hexo new post_name，在source/_posts中会生成文章post_name.md和同名文件夹post_name。将图片资源放在post_name中，文章中引用：

分割线

三个减号（-）或（*）都可以画出一条水平分割线

列表

Markdown支持有序列表和无序列表。
无序列表：直接在文字前面加星号（*）、加号（+）或减号（-）作为列表标记，符号与文字之间有一个字符的空格。在符号前加入两个空格键或者一个tab键就可以产生一个子列表。

有序列表：使用数字接着一个英文句点,

例如你将2. Second 标记为5. Second,其结果都是相同的，为2. Second

反斜杠

\ 相当于反转义作用，使符号成为普通符号。
Markdown支持一下符号前面加上反斜杠来帮助插入普通的符号： 

\ 反斜线
` 反引号
* 星号
_ 底线
{} 花括号
[] 方括号
() 括号
# 井字号
+ 加号
- 减号
. 英文句号
! 惊叹号
表格

使用 | 来分隔不同的单元格，使用 - 来分隔表头和其他行：

name age
lyc 25

为了美观，可以使用空格对齐不同行的单元格，并在左右两侧都使用 | 来标记单元格边界：

name age
lyc 25

为了使 Markdown 更清晰，| 和 - 两侧需要至少有一个空格（最左侧和最右侧的 | 外就不需要了）。

在表头下方的分隔线标记中加入 :，即可标记下方单元格内容的对齐方式：

如果不使用对齐标记，单元格中的内容默认左对齐；表头单元格中的内容会一直居中对齐（不同的实现可能会有不同表现）。
